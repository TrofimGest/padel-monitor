**Итог После Всех Прогонов**

Главный вывод: **строить не браузерного кликера, а скучный HTTP-first мониторинг**.  
`browser-harness` подтвердил HTTP-разведку: данные списков и карточек доступны без JS-рендера. Браузер нужен для разведки, скриншотов, проверки UI-фильтров и ремонта адаптеров.

**Verdict MVP**
Запускать MVP так:

1. **Megapolis**: `HTTP -> HTML parser`.
2. **Realt**: `HTTP -> parse __NEXT_DATA__`.
3. **Kufar**: опционально и осторожно: `SSR HTML / __NEXT_DATA__`, либо API только после явного принятия robots/terms риска.
4. **Playwright/browser-harness**: не cron-основа, а инструмент диагностики.
5. **Vision**: не нужен для цены, площади, адреса, даты.

**Стратегии**

| Стратегия | Плюсы | Минусы | Когда использовать |
|---|---|---|---|
| HTTP parser | Самый простой cron | Нужны селекторы | Megapolis |
| HTTP + embedded JSON | Стабильнее DOM | Может меняться схема | Realt, Kufar |
| Playwright headless | Видит JS UI | Тяжелее, больше bot-риск | Только fallback |
| browser-harness | Реальный Chrome, скрины, DOM/network | Не production-runner | Разведка/ремонт |
| VPS | Автономно | IP-риск | Megapolis/Realt после тестов |
| Local PC cron | Меньше IP-рисков | ПК должен быть включен | Первый тестовый период |
| Browser Use Cloud | Изоляция | Дороже/сложнее | Не нужен для MVP |

**Реальные Проблемы**
- `Kufar`: API удобный, но `robots.txt` запрещает многие query/cursor/sort URL. Делать осторожно.
- `Realt`: дефолтная сортировка “Рекомендуемые”, не чистая новизна. Нужны `createdAt/updatedAt`.
- `Megapolis`: HTML шумный, нужен DOM parser, не regex по всему тексту.
- Дубли между сайтами почти неизбежны.
- Контакты/телефоны лучше не собирать и не раскрывать автоматически.
- Фото не скачивать пачками, хранить URL/превью.

**Четкое Решение**
- **На ПК:** первые 7-10 дней гонять весь мониторинг, плюс `browser-harness` для диагностики.
- **На VPS:** переносить сначала Megapolis и Realt.
- **Kufar:** включать отдельным флагом, без deep pagination и без телефонов.
- **Playwright:** только если HTTP перестал давать данные.
- **Vision:** не использовать.
