# Госаукционы аренды — au.nca.by (site spec)

Статус: **РАБОТАЕТ** (разведка + адаптер 2026-07-05).
Адаптер `padel_monitor/adapters/nca_auctions.py`, source `nca-auction`,
включён (`sources.nca_auctions.enabled: true`). Ценность высокая: госимущество
сдаётся часто СИЛЬНО дешевле рынка (в выдаче — 973 м² со стартом 1949 BYN ≈
2 BYN/м² против рыночной медианы ~18), попадаются «Универсальный спортивный
зал», «зрительный зал», «физкультурно-оздоровительное помещение» — идеальны
под падел.

## ⚠️ Требование: белорусский IP (split-tunnel)

au.nca.by отдаёт данные ТОЛЬКО на BY-IP. Не с BY-адреса — **403** → адаптер
бросает `AdapterStop`, остальные источники работают. На машине разработки за
Amnezia WG настроен split-tunnel: host-маршрут к IP au.nca.by мимо туннеля,
через физический BY-шлюз. Так весь агентский трафик остаётся в VPN, а запросы
к au.nca.by уходят с реального белорусского IP.

Настройка маршрута (macOS, IP au.nca.by = 93.125.23.82, физшлюз 192.168.99.1):

```
sudo route -n add -host 93.125.23.82 192.168.99.1     # добавить
sudo route delete 93.125.23.82                         # убрать
route -n get 93.125.23.82 | grep interface             # проверить (должен en0, не utun)
```

Маршрут непостоянный (исчезает при перезагрузке). Для production на BY-VPS
split-tunnel не нужен — там IP и так белорусский. IP au.nca.by может смениться:
`dig +short au.nca.by` и обновить маршрут. Механизм на dev-машине уже
используется для самого VPN-сервера (прецедент в таблице маршрутов).

## Устройство сайта (jQuery/Spring, ЕРИ)

- Сессия: GET `/` → в HTML скрытые токены `as_sfid`, `as_fid` + cookies.
- Поиск: POST `/mainSearch` (→ 302 → GET `/search`), критерии хранятся в сессии.
  Параметры: `searchItem.howToUseAction=1` (Сдача в аренду; `1001` = продажа
  права аренды), `searchItem.type=5` (Помещение), `squareMin`/`squareMax`
  (серверный фильтр площади — РАБОТАЕТ), `pageSize` (до 500 — все лоты одним
  запросом), `pageNum`. Регион (`address.ateObject`) серверно НЕ применяется →
  Минск отбираем по адресу карточки.
- Карточка выдачи (`.card` с `a[href^=/item/]`): тип, `Площадь: N`, `Адрес`, у
  лотов с назначенным аукционом — дата и `Цена лота: N BYN`.
- Detail `/item/{id}`: инфо-грид `.info-row-title`→значение, фото `/img/ergi/*`,
  полное назначение. Высота потолков почти не встречается → height_unknown,
  уточняется по фото через `padel-enrich` (`nca-auction` в PARSERS).

## Особенности данных (важно для судьи)

- **Площадь у мультиюнитовых зданий = площадь ВСЕГО здания** (Минск-Арена,
  Дворец Республики — десятки под-лотов с одной площадью). Адаптер схлопывает
  такие группы (площадь+улица) в один представитель, предпочитая спорт/зал-тип.
  Судья предупреждён: реальную площадь помещения проверять на au.nca.by/по фото.
- **Цена = начальная ставка аукциона**, не готовая аренда. `price_per_m2` не
  считаем → рыночную медиану аукционы не портят.
- Много лотов «предлагается» (без даты торгов — открытое предложение аренды) +
  немного с назначенной датой (`⏰ торги`). Оба типа полезны.
- В отчёте — отдельная секция `auctions` (не мешаются с готовой арендой).

## Стоп-сигналы

403 (не BY-IP) / 429 / нет токенов / нет карточек в выдаче → `AdapterStop`.
